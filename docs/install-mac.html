Open Viboplr the first time

Viboplr is open-source and free, so it isn't signed with a paid Apple Developer certificate. macOS shows an extra warning the very first time you open it — here's how to get past it. You only do this once; every launch and update after that is normal.

  1. Open the disk image

    Double-click the downloaded Viboplr_…_aarch64.dmg (usually in your Downloads folder), then drag the Viboplr icon onto the Applications folder in the window that appears.

  2. Launch it from Applications

    Open Applications in Finder and double-click Viboplr. macOS will refuse to open it and show one of two messages — pick the matching step below.

  3. If it says "cannot be opened because the developer cannot be verified"

    Control-click (or right-click) the Viboplr icon in Applications, choose Open, then click Open again in the dialog. That's it — Viboplr opens and remembers your choice.

    On macOS Sequoia (15) and later the right-click may not offer Open. Instead go to System Settings → Privacy & Security, scroll to the bottom, and click Open Anyway next to the Viboplr message.

  4. If it says "Viboplr is damaged and can't be opened"

    Despite the wording, the app is not damaged — this is macOS's quarantine flag on downloads from unidentified developers (common on Apple Silicon). Clear it in one command: open Terminal (Applications → Utilities), paste the line below, press Return, then open Viboplr normally.

    xattr -cr /Applications/Viboplr.app

    If macOS asks for your password or permission for Terminal to control Finder, that's expected — approve it and re-run the command.

Why the warning? Apple charges a yearly fee for the certificate that would make this prompt disappear. Viboplr skips it to stay free — the app is fully open-source (read the code on GitHub) and downloads only from GitHub Releases. Once opened, Viboplr updates itself quietly in the background with no repeat of these steps.
